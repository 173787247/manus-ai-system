# 用户使用指南

## 🚀 快速启动

### Windows用户

```bash
# 方法1: 双击运行
start.bat

# 方法2: 命令行运行
python main.py
```

### Linux/Mac用户

```bash
# 方法1: 使用脚本
chmod +x start.sh
./start.sh

# 方法2: 直接运行
python main.py
```

## 📱 访问系统

启动后，在浏览器中访问：

**http://localhost:7860**

如果浏览器没有自动打开，请手动访问上述地址。

---

## 🎯 如何使用

### 步骤1: 输入任务

在"任务指令"文本框中输入您想要执行的任务，例如：

```
打开记事本，输入"Hello World"，保存为test.txt
```

### 步骤2: 设置参数（可选）

- **最大步骤数**: 设置任务执行的最大步骤数（1-50）
- 默认值: 10

### 步骤3: 执行任务

点击 **"执行任务"** 按钮

### 步骤4: 查看结果

在"执行结果"区域查看任务执行结果

## ⚠️ 注意事项

### 1. API密钥配置（可选）

如果没有配置API密钥，系统会使用模拟模式运行：
- 任务分解功能正常
- 知识检索功能正常
- GUI操作功能正常（但不会实际执行）
- 代码生成功能正常（但不会实际执行）

### 2. GUI操作

GUI操作需要：
- 系统有图形界面
- 允许程序控制鼠标键盘
- Windows系统可能需要管理员权限

### 3. 任务描述

- 尽量使用清晰、具体的描述
- 包含具体的操作步骤
- 指定文件路径和名称

## 🐛 常见问题

### Q1: 无法启动系统

**解决**:
```bash
# 检查Python版本
python --version  # 应该是 3.9+

# 安装依赖
pip install -r requirements.txt
```

### Q2: 浏览器无法访问

**解决**:
1. 检查端口7860是否被占用
2. 尝试访问 http://127.0.0.1:7860
3. 检查防火墙设置

### Q3: 任务执行失败

**可能原因**:
- 任务描述不够清晰
- 需要GUI操作但系统不支持
- API密钥未配置（某些功能需要）

**解决**:
- 使用更清晰的任务描述
- 检查系统状态
- 查看错误信息

### Q4: 如何停止系统

**方法**:
- 在运行窗口按 `Ctrl+C`
- 关闭命令行窗口

## 💡 使用技巧

### 1. 任务描述技巧

✅ **好的描述**:
```
打开记事本，输入"Hello World"，保存为C:\Users\test.txt
```

❌ **不好的描述**:
```
做点事情
```

### 2. 分步执行

对于复杂任务，可以分解为多个简单任务：
1. 先执行：打开记事本
2. 再执行：输入内容
3. 最后执行：保存文件

### 3. 查看执行详情

执行结果会显示：
- 任务状态（成功/失败）
- 执行步骤数
- 执行详情
